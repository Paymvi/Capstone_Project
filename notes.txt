Before making changes always do "git branch" to make sure you are on the right branch

Doing git status is similar to looking at the git tab on the vscode

To keep your changes but remove the commit:
git reset --soft HEAD~1


Conventional Commits:

    feat: → new feature
    fix: → bug fix
    refactor: → code restructuring
    style: → formatting / CSS
    docs: → documentation
    chore: → setup/config changes


How to rebase:
    git branch
    git switch new-branch
    git fetch origin

    *Make sure you've commited and pushed to the branch*
    git rebase origin/main

    *remove conflicts* then git add .
    git rebase --continue

    git push --force
    OR
    git push --force --set-upstream origin deposit

    If you mess up:
        git rebase --abort
